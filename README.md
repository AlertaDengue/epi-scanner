# Dengue Epi-Scanner App

This H2O Wave application analyzes the expansion of dengue and chikungunya in
Brazil using up-to-date incidence data from Infodengue.

[![watch the video](https://img.youtube.com/vi/LQmMhVWVJUs/hqdefault.jpg)](https://youtu.be/LQmMhVWVJUs)

## Getting Started

## Run with Miniforge environment

### Pre-requisites

- [Miniforge](https://github.com/conda-forge/miniforge) installed
- [Poetry](https://python-poetry.org/) installed

```bash
# Install dependencies and activate the conda environment
mamba env create -f conda/env-base.yaml
conda activate episcanner

# Install project dependencies (including dev tools such as makim)
poetry install --with dev
````

## Populate the `data/` directory

```bash
python ./epi_scanner/management/cli.py [disease]
```

Then start the app:

```bash
wave run --no-reload --no-autostart epi_scanner.app
```

## Running with containers (Makim)

### Pre-requisites

* [Docker](https://docs.docker.com/get-docker/) installed and running
* [Miniforge](https://github.com/conda-forge/miniforge) installed
* [Poetry](https://python-poetry.org/) installed

Makim is used to standardize helper commands for local, CI, and production
container workflows.

### Start the app (dev profile)

```bash
# Start/build the container stack for the Wave app
makim episcanner.deploy --profile dev --service wave
```

### Wait for the service to become healthy

```bash
makim episcanner.container-wait --service wave
```

### Stop and remove containers/volumes

```bash
makim episcanner.down --profile dev --service wave
```

> Tip: You can replace `dev` with `ci` or `prod` depending on your target
> environment.

## Data Update

The data update is now generated by the *epi_scanner-downloader* tool.

For more information, please see the
[epi_scanner-downloader repository](https://github.com/AlertaDengue/episcanner-downloader/tree/main/epi_scanner).
